<task type="auto">
  <name>Task 1: SonumScraper with Bitrix pagination and HTML extraction</name>
  <files>src/lineaf/scrapers/sonum.py, src/lineaf/run_scrapers.py</files>
  <action>
    Create `src/lineaf/scrapers/sonum.py` with class `SonumScraper(BaseScraper)`:

    **Constructor:**
    - `super().__init__(site_name="sonum", catalog_url="https://www.sonum.ru/catalog/matrasy/?filter%5Bwidth%5D%5B0%5D=160&filter%5Blength%5D%5B0%5D=200")`

    **`collect_product_urls(self, page) -> list[str]`:**
    - Iterate pages: append `&PAGEN_1={page_num}` to catalog_url, starting at page 1.
    - On each page: use Playwright `page.query_selector_all()` to find product card elements. Try selectors in order: `.catalog-item a[href]`, `.product-card a[href]`, `a[href*="/catalog/matrasy/"]`. Use the first selector that returns results.
    - Extract `href` attribute from each product link. Build absolute URL: if href starts with `/`, prepend `https://www.sonum.ru`.
    - Deduplicate URLs (same product may appear in multiple cards).
    - Stop pagination when page returns 0 product cards.
    - Call `self.delay()` between pages.
    - Return full list of absolute URLs.

    **`extract_product(self, page, url: str) -> dict`:**
    - Navigate with `self.goto_with_retry(page, url)`.
    - **Name:** Try `page.query_selector("h1")` -> `inner_text()`. Fallback: `page.title()`.
    - **Prices:**
      - Try to find price elements. Look for elements containing "₽" or price-like patterns.
      - Try selectors: `.product-price`, `.price`, `[class*="price"]`.
      - Parse all price-like text found. The lower price is sale, the higher is original. If only one found, it's the sale price.
      - Use `parse_price()` from utils.
    - **Characteristics table:**
      - Look for `<table>` or `<dl>` elements containing product specs.
      - Try selectors: `.product-chars table tr`, `.product-info table tr`, `table.chars tr`, `.characteristics tr`.
      - For each row: extract label (first cell/dt) and value (second cell/dd).
      - Build characteristics dict from label->value pairs.
    - **Field mapping (Russian labels from RESEARCH.md):**
      - `firmness` <- "Жесткость матраса" or "Жесткость"
      - `height_cm` <- "Высота матраса" or "Высота"
      - `cover_material` <- "Материал чехла"
      - `weight_kg` <- "Вес на 1 место" or "Вес"
      - `filler` <- "Наполнитель" (may not be in characteristics table — try description text as fallback via regex `r"[Нн]аполнитель[:\s]+([^.]+)"` in page text. Store NULL if not found — per user decision this is OK.)
    - Return dict with all fields. Optional fields default to None.
    - Call `self.delay()` after extraction.

    **Error handling:**
    - If h1 not found -> log warning, try title tag.
    - If no prices found -> log warning, return dict with name only (validate_product will skip it).
    - If characteristics table not found -> log info, all optional fields will be None.

    **Register in run_scrapers.py:** Add `from lineaf.scrapers.sonum import SonumScraper` to the spider registry.

    NOTE: CSS selectors have MEDIUM confidence from research. The spider should try multiple selector fallbacks. If ALL selectors fail, log the page HTML snippet (first 500 chars) for debugging.
