  <action>
    1. Create `src/lineaf/dashboard/__init__.py` (empty).

    2. Create `src/lineaf/dashboard/app.py` — full Streamlit application:

    **Page config:**
    - `st.set_page_config(page_title="Lineaf — Цены конкурентов", layout="wide")`
    - `st.title("Lineaf — Цены конкурентов")`
    - API_BASE = `os.environ.get("API_BASE_URL", "http://localhost:8000/api")`

    **Helper functions (cached):**
    - `@st.cache_data(ttl=60)` on all API fetch functions to avoid re-fetching on every widget interaction
    - `fetch_prices(sites)` -> calls GET /api/prices with site params
    - `fetch_price_history(product_id)` -> calls GET /api/prices/history
    - `fetch_price_index()` -> calls GET /api/prices/index
    - `fetch_runs()` -> calls GET /api/runs
    - `fetch_freshness()` -> calls GET /api/runs/freshness
    - `fetch_changes()` -> calls GET /api/products/changes

    **Sidebar:**
    - `st.sidebar.header("Фильтры")`
    - Multiselect "Конкурент" with options ["askona", "ormatek", "sonum"], format_func to display Russian names ("Аскона", "Орматек", "Сонум"), default=all
    - Site name mapping dict: `{"askona": "Аскона", "ormatek": "Орматек", "sonum": "Сонум"}`

    **Freshness indicator (above tabs):**
    - Call fetch_freshness(). Display 3 columns, one st.metric per site.
    - Use `:red[site_name]` or `:green[site_name]` in metric label based on is_stale.
    - Value = formatted date (DD.MM.YYYY HH:MM) or "Нет данных" if null.

    **KPI row (above tabs):**
    - Call fetch_price_index(). Display st.metric per competitor: "Средняя цена: X руб."

    **Tab 1: Цены (DASH-01, DASH-03)**
    - Load prices via fetch_prices(selected_sites).
    - Convert to pandas DataFrame.
    - Display with `st.dataframe(df, use_container_width=True)` — built-in sort/search.
    - Add text_input for name search filter (filter df before display).
    - Export section: `st.download_button("Экспорт в Excel", ...)` — call GET /api/export, pass response content as data. File name: "lineaf_prices.xlsx".
    - Also offer CSV via `df.to_csv()` with `st.download_button("Экспорт в CSV", ...)`.

    **Tab 2: Графики (DASH-02)**
    - Three sub-sections via st.subheader:

    2a. "Динамика товара":
    - Selectbox to pick a product (from the prices list). Display product name.
    - Fetch history via fetch_price_history(product_id).
    - px.line chart: x=scraped_at, y=price_sale, title="Динамика цены"
    - `st.plotly_chart(fig, width="stretch")` — NOT use_container_width (deprecated).

    2b. "Сравнение конкурентов":
    - Fetch all price history grouped by site (use prices data, group by source_site + scraped_at, compute mean).
    - px.line: x=scraped_at, y=avg_price_sale, color=source_site
    - Russian labels via labels= parameter.

    2c. "Распределение цен":
    - Use current prices data.
    - px.box: x=source_site, y=price_sale, color=source_site
    - Russian labels.

    **Tab 3: Изменения (DASH-06)**
    - Call fetch_changes().
    - Two columns: "Новые товары" and "Удалённые товары".
    - Display as st.dataframe or st.table for each list. Show name and site (Russian name).
    - If lists are empty, show st.info("Нет изменений за последний период").

    **Tab 4: Логи (SCHD-02)**
    - Call fetch_runs().
    - Display as st.dataframe with columns: site (Russian name), status, started_at, finished_at, products_found, products_new, products_removed, error_message.
    - Color-code status: use conditional formatting or just display as-is (Streamlit dataframe has limited formatting).
    - "Запустить парсинг" button: on click, POST /api/scrape. Show st.success("Парсинг запущен в фоне") or st.error on failure.

    ALL labels, headers, buttons, tooltips in Russian per user decision.
    Use `requests` library for API calls (simpler than httpx for sync Streamlit).
