  <name>Task 1: Initialize uv project with all dependencies and Docker Compose</name>
  <files>pyproject.toml, .python-version, .gitignore, .env.example, .env, docker-compose.yml</files>
  <action>
1. Run `uv init` in the project root (or create pyproject.toml manually if uv init conflicts with existing files). Set project name to `lineaf-parser`, version `0.1.0`, requires-python `>=3.11`.

2. Add production dependencies:
   `uv add sqlalchemy alembic psycopg2-binary pydantic-settings python-dotenv fastapi "uvicorn[standard]"`

3. Add dev dependencies:
   `uv add --dev pytest`

4. Create `.python-version` with content `3.11` (if uv init didn't create it).

5. Create `.gitignore` with standard Python entries plus `.env`, `__pycache__`, `.venv`, `*.pyc`, `.pytest_cache`, `postgres_data/`.

6. Create `.env.example`:
   ```
   DATABASE_URL=postgresql://lineaf:lineaf@localhost:5432/lineaf_dev
   TEST_DATABASE_URL=postgresql://lineaf:lineaf@localhost:5432/lineaf_test
   DEBUG=false
   ```

7. Create `.env` (copy of .env.example with same values for local dev — gitignored).

8. Create `docker-compose.yml` with PostgreSQL 16 Alpine:
   - Service name: `db`
   - Image: `postgres:16-alpine`
   - Env: POSTGRES_USER=lineaf, POSTGRES_PASSWORD=lineaf, POSTGRES_DB=lineaf_dev
   - Ports: 5432:5432
   - Volume: `postgres_data:/var/lib/postgresql/data`
   - Healthcheck: `pg_isready -U lineaf -d lineaf_dev` every 5s, timeout 5s, retries 5
   - Add an `init-test-db` service that runs `psql` to CREATE DATABASE lineaf_test IF NOT EXISTS (so test DB is auto-created on first `docker compose up`).
  </action>
  <verify>
    <automated>uv sync --frozen 2>&1 | tail -5 && python -c "import sqlalchemy; import alembic; import pydantic_settings; print('deps OK')"</automated>
  </verify>
  <done>pyproject.toml exists with all dependencies, uv.lock is generated, docker-compose.yml defines PostgreSQL with healthcheck, .env and .env.example both exist</done>
